# Wellbeing Analyzer + Post API 테스트 방법

## 🚀 빠른 테스트 가이드

### 1단계: Django 서버 실행

**터미널 1 (PowerShell 또는 CMD):**
```bash
cd C:\Users\thddl\PhotoBlog
.\venv\Scripts\Activate.ps1
python manage.py runserver
```

**확인:** 브라우저에서 `http://127.0.0.1:8000` 접속해서 서버가 실행 중인지 확인

---

### 2단계: Wellbeing Analyzer 실행

**터미널 2 (새로운 PowerShell 또는 CMD 창):**
```bash
cd C:\Users\thddl\PhotoBlog
.\venv\Scripts\Activate.ps1
python wellbeing_analyzer.py
```

**확인:**
- 웹캠 창이 열리고 실시간으로 객체 검출이 시작됩니다
- 화면에 검출된 객체들이 박스로 표시됩니다

---

### 3단계: Post 게시 확인

#### 방법 1: 브라우저에서 확인
1. 브라우저 열기
2. 주소창에 입력: `http://127.0.0.1:8000/api_root/Post/`
3. 게시된 포스트 목록 확인

#### 방법 2: 콘솔 메시지 확인
터미널 2에서 다음 메시지들이 나타나면 성공:
- `🆕 새 객체 검출: person` (새 객체가 검출될 때)
- `✅ Post 게시 성공: Detected Objects: ...` (게시 성공 시)

---

## 📋 예상 동작

### 정상 동작 시:
1. 웹캠 창이 열림
2. 사람이나 물건이 검출되면 박스로 표시됨
3. 새 객체가 검출되면 콘솔에 메시지 출력
4. 5분마다 또는 새 객체 검출 시 Post 자동 게시

### 문제 발생 시:
- `❌ Post 게시 오류: ...` 메시지가 나타나면 Django 서버가 실행 중인지 확인
- 웹캠이 열리지 않으면 웹캠이 연결되어 있는지 확인

---

## ⚙️ 설정 변경 (선택사항)

`wellbeing_analyzer.py` 파일을 열어서 상단 설정 부분 수정:

```python
# Post 게시 비활성화하려면
ENABLE_POST_API = False

# 게시 주기를 변경하려면 (초 단위)
POST_INTERVAL = 60  # 1분마다 게시

# 새 객체 검출 시 즉시 게시를 끄려면
POST_ON_NEW_OBJECT = False
```
